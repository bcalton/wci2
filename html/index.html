Welcome

This is the home of the eartH2Observe Water Cycle Integrator (WCI). The WCI portal takes data that you select and plots it on a map to help you analyse, export and share it.

How to use the WCI Portal

It's still in development

It's important to remember that this version is an early version and still in development. You will discover things that do not work properly or behave in a different way to the way you expected. Please bear this in mind

The right browser

The WCI portal has been built to take advantage of the capabilities of modern browsers so you will need the latest version of Firefox or Chrome, or versions 9, 10 or 11 of Internet Explorer. You will also need to have Javascript enabled

Creating a map

Select some indicators - when the portal first loads you will see a layer showing the base map overlayed with country borders. The panel on the left of the screen enables you to select indicators that are of interest by clicking on the icon to the right of the indicator name (looks like an eye). A list of popular indicators is displayed first, however, you can select from drop down lists of indicators grouped by type of by data source. Click on the 'Search' tab to find an indicator using the name or part of the name.

Build a map - once you have selected the required indicators, click the 'Build a Map' button to view further information about the indicators you have selected.

Navigate - there will be a set of buttons that allows you to navigate around the map and to zoom in and out, however, for now you will have to use your mouse. Click and hold the left mouse button to drag the map around and use the scroll wheel to zoom in and out.

Analyse - in a future release you will be able to select a bounding box for an area of interest and create a timeseries plot for the selected indicators, or export these data for external processing. You will find that controls exist for these features, however, at present they are not functioning.

Development & Feedback

Milestone 3 of the eartH2Observe project is set for June 2014 and this requires the first version of the data portal to be available online. The portal development will continue throughout the duration of the project; milestone 8 in September 2015 marks the halfway point, and deliverable D7.9 in November 2017 is the completed version of the portal.

With each release of the portal a list of enhancements made and bugs fixed will be published here so that all parties can see progress as it unfolds.

Ideas and Feedback - if you have ideas for how the portal should operate we would really like to hear them. In fact, the sooner you are able to offer feedback on existing functionality and/or suggest ideas for enhancements the greater the chance of them being included. If you come up with a fantastic idea in October 2017 there won't a lot of time to develop it.

Please send your ideas/suggestions/feedback/(complaints) to Ben Calton at PML

Data Services

EartH2Observe is working to integrate available earth observations, in-situ datasets and models, in order to construct a consistent global water resources reanalysis. The resulting and intermediate datasets will be made available here through a variety of methods:

  • The Water Cycle Integrator data portal
  • Open and freely accessible data services - OPeNDAP, WMS and WCS available using THREDDS Data Server: http://wci.earth2observe.eu/thredds
  • A data exchange (primarily for internal project use)
    • FTP server: wci.earth2observe.eu
    • Partner Access: user id and password as supplied
    • Guest Access: e2o_guest / oowee3WeifuY1aeb (limited read-only access)

About the Portal

The WCI portal is an open source project built by Plymouth Marine Laboratory's Remote Sensing Group. The portal builds on the development of several other EU funded projects, past and present, that PML have involvement in. You can find the code on GitHub at https://github.com/earth2observe-pml/GISportal

The frontend views is HTML and Javascript and uses OpenLayers to display map layers produced by THREDDS, an OGC standards Web Map Service. There is a middleware section written in Python that accepts AJAX requests initiated by the user/browser and transforms these requests into WMS and WCS requests.
